# 成本管理看板 - 需求文档（V4 交互与视觉优化版）

> 本版本聚焦老板核心关切——**成本管控**，明确信息优先级，并在 UI/UX 层面结合现代设计规范，提供高效、直观的管理工具。

---

## 一、项目背景与核心目标

### 1.1 当前阶段目标
根据老板口头需求，快速生成一个看板页面，用于**验证产品经理对业务的理解是否正确**。看板需让老板能够：
1. 清楚掌握部门经营情况
2. 便于进行部门管理

### 1.2 业务背景
- **公司性质**：政府 IT 服务商
- **服务对象**：四川省、成都市及下辖 23 个区县、部分街道等政府单位
- **主营业务**：为上述单位提供信息化建设及运营服务
- **部门规模**：23 人（内部人力固定）
- **项目数量**：约 30 个项目

### 1.3 核心痛点（老板最关心）
> **“成本怎么花的？花去哪些类目了？是不是在限定的额度内？”**

老板当前最迫切的需求是**成本管控**：
- 清楚每一笔成本的去向（按类目拆解）
- 实时监控成本是否超出预算/限额
- 快速定位超支项目或成本项

### 1.4 信息优先级
| 优先级 | 模块 | 老板关注点 |
|--------|------|------------|
| **P0** | 成本总览与结构 | 四大类成本实际 vs 计划、超支预警 |
| **P1** | 项目成本明细 | 哪些项目超支、超在哪个类目、关联销售/PM/区县 |
| **P2** | 收入与利润概览 | 整体经营健康度（计划 vs 实际） |
| **P3** | 辅助查询 | 按销售/项目经理/区县筛选，未来扩展贡献值 |

## 二、成本管控逻辑详解

### 2.1 成本四大类目
| 类目 | 说明 | 管理属性 |
|------|------|----------|
| **内部人力** | 23 名固定员工的工资（老板、产品、运营、售前、销售、项目经理等） | 固定成本，整体展示简化；项目层面需记录人员投入，为未来 ROI 分析打基础 |
| **外采人力** | 项目需要时额外申请的研发、测试等外部人员费用 | 可变成本，严格监控，通常设项目级限额（如 ≤ 项目金额的 10%） |
| **商务费用** | 销售请客吃饭、送礼等 | 可变成本，需监控 |
| **其他** | 杂费（打车、打印）、媒体宣传费、招投标专家费等 | 可变成本，暂不细分，统一归入 |

### 2.2 管控指标（额度）
- **整体层面**：部门年度利润率 ≥ 20%（示例，可配置）
- **项目层面**：
  - 单个项目利润率 ≥ 30%（示例，可配置）
  - 单个项目外采成本 ≤ 项目金额的 10%（示例，可配置）
  - 其他可变成本（商务+其他） ≤ 项目金额的 5%（示例，可配置）
- **非项目成本**：部门整体商务费用、其他费用需控制在一定额度内（单独监控）

### 2.3 预警机制
- 当实际成本超过计划额度或达到阈值时，对应位置显示红色预警标识
- 预警分为**整体预警**（部门总成本超支）和**明细预警**（项目/类目超支）
- 超支项目自动高亮，支持快速下钻

## 三、整体架构与交互基础

### 3.1 技术架构
- **SPA 单页面应用**：通过 JavaScript 控制不同维度（总览/项目/销售）的内容切换，配合 CSS 实现页面切换的淡入淡出效果（opacity + transition）。

### 3.2 侧边栏
- 深蓝色背景（#1a2740），宽度 240px（收起时 64px）
- 支持折叠/展开，折叠时菜单项仅显示图标，hover 显示 Tooltip
- 当前所在菜单项高亮：左侧 4px 蓝色条（#4080FF） + 背景微亮（rgba(255,255,255,0.1)）

### 3.3 内容区
- 浅灰背景（#F8FAFC），内边距 24px
- 所有卡片采用 **16px 圆角** + 柔和阴影（box-shadow: 0 2px 12px rgba(0,0,0,0.04)），悬停时阴影加深（0 4px 16px rgba(0,0,0,0.08)）

### 3.4 色彩体系
- **主色**：蓝青渐变（如按钮、关键数据使用 #4080FF → #14C9C9 渐变）
- **状态标签**：
  - 进行中 → 蓝色（#4080FF）
  - 已签约 → 绿色（#3DC779）
  - 待评估 → 黄色（#FAC858）
  - 已完成 → 灰色（#9CA3AF）
- **预警色**：
  - 红色（#EE6666）用于超支/不达标
  - 黄色（#FAC858）用于接近阈值
  - 绿色（#3DC779）用于达标

## 四、功能模块与交互设计（按优先级）

### 4.1 P0：成本总览与结构

#### 4.1.1 核心指标卡片（精简版）
三个卡片横向排列：
- **总成本**：展示实际总成本 vs 计划总成本，完成率，进度条
- **可变成本合计**：外采人力 + 商务费用 + 其他，实际 vs 计划，完成率，进度条
- **固定成本（内部人力）**：仅展示实际金额，无计划对比（简化）

**交互与动画**：
- 卡片依次出现（stagger 0.05s），伴随轻微上浮效果
- 数字使用 32px 加粗，对比信息使用 12px 次要色
- 进度条高度 8px，圆角 4px，填充色根据完成率动态变化（达标绿，超支红）

#### 4.1.2 成本结构图（环形图）
- 使用 ECharts 环形图，中心显示总成本金额
- 配色：固定成本（#E5E7EB），外采人力（#4080FF），商务费用（#FAC858），其他（#EE6666）
- **动画**：图表加载时执行 ECharts 默认的扇形展开、数字渐变
- **交互**：悬停扇形时高亮并显示具体金额与占比，图例支持点击显示/隐藏对应类目

#### 4.1.3 成本分类明细表
| 成本类型 | 计划金额 | 实际金额 | 差异 | 完成率 | 进度条 |
|----------|----------|----------|------|--------|--------|
| 内部人力 | - | ✓ | - | - | - |
| 外采人力 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 商务费用 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 其他 | ✓ | ✓ | ✓ | ✓ | ✓ |

**交互**：
- 表格支持排序（按实际金额、差异绝对值）
- 进度条使用圆角细条（高度 6px），完成部分使用蓝色渐变，超支部分使用红色渐变
- 差异值正数（超支）显示红色，负数（节约）显示绿色，并带 +/- 符号

#### 4.1.4 整体利润预警
- 若当前部门实际利润率低于计划利润率，顶部全局提示条显示预警（带红色背景和警告图标）

---

### 4.2 P1：项目成本明细

#### 4.2.1 筛选区
- **状态筛选**：Segmented 控件（全部 / 进行中 / 已签约 / 待评估 / 已完成），选项后显示数量徽章，切换时内容区重新加载表格（淡入淡出过渡）
- **搜索框**：实时搜索（防抖 300ms），无结果时显示空状态插画
- **多维度筛选**：下拉多选（销售/项目经理/区县），支持清空，筛选后表格平滑更新

#### 4.2.2 项目列表表格
| 字段 | 说明 | 样式增强 |
|------|------|----------|
| 项目名称 | 含区县信息 | 主文字加粗，区县信息 12px 灰色 |
| 状态 | Tag | 按状态配色 |
| 签约金额 | actualAmount | 右对齐，可排序 |
| 总成本 | actualCost | 右对齐，可排序 |
| 外采成本占比 | 外采/签约金额 | 若超阈值（如>10%）显示红色背景 + ⚠️ 图标 |
| 利润率 | 实际利润率 | 若低于阈值（如30%）显示红色背景 + ↓ 图标 |
| 进度 | progress | 进度条（高6px，圆角，蓝色填充）+ 百分比 |
| 销售/项目经理 | 双行显示 | 销售（14px）、项目经理（12px 灰色） |
| 操作 | 详情 | 点击弹出抽屉（Drawer） |

**表格交互**：
- 固定表头，支持横向滚动（当列过多时）
- 行高 56px，悬停背景 #F9FAFB
- 点击列头可排序（签约金额、总成本、利润率、进度），排序状态图标明确（↑↓）

#### 4.2.3 项目详情抽屉（Drawer）
- 点击“查看详情”按钮或行末操作图标，右侧滑出抽屉，宽度 480px，背景半透明遮罩
- 抽屉内容：
  - **成本明细**：展示该项目四大类成本的实际金额、计划金额、差异
  - **人员投入（预留）**：以表格或时间轴形式展示项目内部人力投入记录（角色、工时、成本分摊），当前使用 Mock 数据，为未来贡献值模块打基础
  - **操作按钮**：关闭、编辑（未来）

### 4.3 P2：收入与利润概览

#### 4.3.1 收入对比图
- 分组柱状图（计划收入 vs 实际收入），支持切换年度（下拉选择）
- **动画**：入场时从 0 增长到实际值，带缓动曲线
- **交互**：悬停显示具体数值和占比，图例支持显示/隐藏系列

#### 4.3.2 利润趋势图
- 折线图：计划利润率 vs 实际利润率（月度/年度）
- 实际利润率低于计划时折线变为红色，并标注预警点（标记点 tooltip）
- 支持图例切换

#### 4.3.3 项目总数与分布
- 卡片形式展示项目总数及各状态数量
- 可选：迷你占比环形图（或直接显示数字）

### 4.4 P3：辅助查询与未来扩展
- 人员投入记录在项目详情抽屉中以表格或时间轴形式展示（Mock 数据），为未来贡献值模块预留接口
- 多维度筛选支持按销售、项目经理、区县筛选项目列表，满足不同管理视角的查询需求

## 五、技术栈与开发环境

| 类型 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | React 19 + Vite 8 | 构建高效 SPA，支持热更新 |
| 语言 | TypeScript | 类型安全，便于维护 |
| UI 组件库 | Semi Design 2.x | 提供高质量的组件，支持主题定制（深蓝主题） |
| 图表 | ECharts 6 + echarts-for-react | 满足所有图表需求，自带丰富动画 |
| 样式方案 | CSS Modules + 全局主题变量 | 保证样式隔离，同时统一颜色/间距变量 |
| 路由 | React Router 6 | 管理总览/项目/销售三个视图的切换 |
| 状态管理 | Zustand / React Context | 轻量级，管理筛选条件、年度切换等全局状态 |
| 数据请求 | Axios（Mock 数据先使用 fetch） | V1 纯前端 Mock，V2 可对接真实接口 |

**开发环境**：
- 启动命令：`cd ~/project/win && npm run dev`
- 访问端口：5688

## 七、交互亮点汇总

1. **页面切换淡入淡出**：CSS transition 实现视图切换平滑过渡。
2. **图表入场动画**：ECharts 自带动画 + 自定义 loading 骨架屏。
3. **表格实时搜索与排序**：前端数据处理，用户无感知延迟。
4. **抽屉式详情**：不跳转页面，保持上下文连贯性。
5. **多维度筛选联动**：筛选条件变更后，表格和图表同步更新，所有数据保持一致。
6. **阈值预警视觉反馈**：红/黄/绿背景 + 图标，一眼定位问题项目。
7. **卡片悬停微动效**：提升界面细腻感。
