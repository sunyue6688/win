## 一、项目概述

### 1.1 项目背景
- **公司性质**：政府 IT 服务商
- **服务对象**：四川省、成都市及下辖 23 个区县、部分街道等政府单位
- **主营业务**：信息化建设及运营服务
- **部门规模**：23 人（内部人力固定）
- **项目数量**：约 30 个项目（存量+新增）

### 1.2 核心目标
让老板能够：
1. 清楚掌握部门经营情况（收入、成本、利润）
2. **重点管控成本**：成本是核心管理抓手，所有可变成本需在限额内
3. 快速定位超支项目、成本异常

### 1.3 版本说明
V7 合并为**单一版本**，包含三个板块：
1. **总览看板** — 部门级成本与经营全景
2. **项目看板** — 项目维度的明细与管理
3. **销售看板** — 销售维度的业绩跟踪

### 1.4 信息优先级
| 优先级 | 模块 | 老板关注点 |
|--------|------|------------|
| **P0** | 总览看板 | 成本总览、可变成本管控、利润率、预警 |
| **P1** | 项目看板 | 项目分布、项目经理绩效、项目明细、成本预警 |
| **P2** | 销售看板 | 销售业绩、目标完成率、利润率 |

## 二、成本管控逻辑

### 2.1 成本四大类目
| 类目 | 说明 | 管理属性 |
|------|------|----------|
| **内部人力** | 23 名固定员工工资 | 固定成本，恒定不变，展示降级 |
| **外采人力** | 项目外部研发/测试等 | 可变成本，严格监控，设项目级限额 |
| **商务费用** | 销售请客、送礼等 | 可变成本，需监控 |
| **其他** | 杂费、媒体宣传、招投标专家费等 | 可变成本，统一归入 |

### 2.2 管控指标
#### 部门层面
- 部门年度计划利润率（可配置，如 20%）
- 外采人力年度总额度
- 商务费用年度总额度
- 其他费用年度总额度

#### 项目层面
- 项目外采成本占比上限（如 15.0%）— 超出即预警
- 项目利润率下限（如 30%）

### 2.3 预警机制
- **部门级预警**：实际利润率低于计划利润率时，总览看板顶部黄色提示条
- **外采成本预警**：总览看板顶部黄色提示条，提示"N个项目外采成本即将超出外采成本线"
- **项目级预警**：项目列表中，外采成本占比超限的项目显示红色百分比 + ⚠️

## 六、数据模型

### 6.1 项目
```typescript
interface Project {
  id: string
  name: string
  district: string
  status: '进行中' | '已签约' | '待评估' | '已完成'
  projectType: '存量' | '新增'
  sales: string
  pm: string
  contractAmount: number        // 签约金额
  receivedPayment: number       // 已回款金额
  progress: number              // 项目进度 (0-100)
  planProfitRate: number        // 计划利润率
  externalHRRatioLimit: number  // 外采占比上限
  actualCost: number            // 总成本
  costBreakdown?: {
    internalHR: number
    externalHR: number
    businessCost: number
    otherCost: number
  }
}
```

```typescript
interface DepartmentOverview {

  projectStats: {
    total: number
    inProgress: number
    signed: number
    pending: number
    completed: number
    existing: number
    newProjects: number
  }
}
```

```typescript
interface PMSummary {
```
