# 现代 SaaS 风看板

## 产品概述

**核心理念：** 一个面向高管的成本管理仪表板，能够即时展现全公司范围内的支出、预算执行情况和财务异常。将复杂的财务数据转化为可操作的顶层洞察。

**目标用户：** C级高管和财务总监，他们需要了解高层级的财务健康状况，而不想陷入细碎的电子表格数据中。

**设备：** 桌面端

**设计方向：** 现代企业级 SaaS。柔和的米灰/灰色背景，清晰圆润的白色卡片，高度语义化的图表配色（红色代表超支，绿色代表节省），确保信息能被快速认知。

**灵感来源：** Ramp, Mercury

---

## 屏幕

- **概览仪表板：** 高层级指标、现金消耗率、一目了然的预算状态。
- **项目成本排名：** 按支出排序的项目榜单，并对比其分配预算。
- **预算告警：** 专门展示当前超预算警告和预测异常的视图。
- **部门分解：** 使用矩形树图和条形图展示部门支出分布。

---

## 关键流程

**查看预算异常：** 高管想了解技术预算为何显示红色警告。

1. 用户在概览仪表板上 -> 看到“技术部门”标记了红色警告图标。
2. 用户点击“技术部门”卡片 -> 导航至部门分解视图，并自动筛选“技术”部门。
3. 查看细项支出，发现“云基础设施”超出预算 24%。

**用途：** 高管财务监督的中央指挥中心。

# 现代 SaaS 风看板 — 设计规范

## 1. 设计原则

- **清晰优先于密度：** 保持充裕的留白（卡片内边距 24px），确保高管能快速扫视并理解关键信息，避免视觉疲劳。
- **语义化色彩：** 颜色直接传达业务状态——红色代表超支/告警，绿色代表节省/健康，蓝色代表中性数据。
- **柔和的企业感：** 使用暖调的米灰背景、圆润的卡片和细腻的阴影，营造稳定、可信赖的现代企业工具氛围。
- **响应式友好：** 核心组件在不同设备宽度下都能提供可用的视图（侧边栏可折叠、表格自适应、移动端改用卡片列表）。

## 2. 色彩系统

| 角色 | 色值 (Hex) | 用途 |
| :--- | :--- | :--- |
| **Primary (主色)** | `#0F172A` | 主导航背景、激活状态、主要按钮、关键文字。 |
| **Background (背景)** | `#F7F5F2` | 页面全局背景（米灰/暖灰）。 |
| **Surface (表面)** | `#FFFFFF` | 卡片、侧边栏、模态框、下拉菜单。 |
| **Text (文本)** | `#334155` | 正文、主要标签。 |
| **Muted (柔和)** | `#94A3B8` | 辅助文本、占位符、非活动状态。 |
| **Border (边框)** | `#E2E8F0` | 卡片边框、表格分隔线、侧边栏分隔线。 |
| **Red (超支/告警)** | `#E11D48` | 负向偏差、预算告警、关键警示图标。 |
| **Green (节省/正向)** | `#10B981` | 正向偏差、低于预算状态、健康指标。 |
| **Blue (信息/图表)** | `#3B82F6` | 图表主色、中性数据点、次要链接。 |

## 3. 字体系统

采用两款几何无衬线字体，营造现代、精准的阅读体验。

- **标题/按钮字体 (Plus Jakarta Sans):**
  - **用途：** 页面主标题、卡片标题、按钮文字、导航菜单。
  - **字重：** `600` (半粗), `700` (粗体)。
  - **字号：** `14px` (按钮), `24px` (KPI数值), `28px`-`32px` (页面标题)。

- **正文字体 (Albert Sans):**
  - **用途：** 正文描述、表格内容、标签、工具提示、辅助信息。
  - **字重：** `400` (常规), `500` (中等)。
  - **字号：** `12px` (小号文本、徽章), `14px` (标准正文)。

## 4. 间距与布局

- **全局边框半径：** 卡片 `16px`，按钮 `8px`，徽章 `4px`，其他小元素 `8px`。
- **基础间距单位：** `4px`。常用间距为 `8px`, `12px`, `16px`, `24px`, `32px`。
- **卡片内边距：** 统一 `24px`（移动端可缩减至 `16px`）。
- **侧边栏宽度：** 桌面端固定 `240px`。
- **内容区最大宽度：** 不设硬限制，但通常留出左右 `24px` 外边距，在大屏幕上居中显示。

## 5. 核心组件库

### 5.1. 指标卡片 (KPI Card)
- **结构：** 背景 `var(--color-surface)`，圆角 `var(--radius-card)`，阴影 `var(--shadow-card)`，边框 `1px solid var(--color-border)`，内边距 `var(--spacing-card)`。
- **布局：** 垂直排列，指标数值 `32px` 标题字体，标签 `14px` 正文字体。
- **附加徽章：** 可选，显示环比变化，红色或绿色文字，搭配 10% 透明度的背景色。
- **交互：** 无特殊悬停效果（保持稳定感）。

### 5.2. 数据表格 (用于项目排名等)
- **布局：** 宽度 100%，边框无，表头背景透明，行高 `56px`。
- **表头：** 文本左对齐，`14px` 正文字体，`500` 字重，颜色 `var(--color-muted)`，底部边框 `1px solid var(--color-border)`。
- **单元格：** 内边距 `0 16px`，垂直居中。数值列右对齐。
- **悬停行：** 背景色变为 `#F8FAFC`（浅灰）。
- **进度条：** 高 `8px`，圆角 `999px`，背景 `#E2E8F0`，填充色动态为 `var(--color-green)` 或 `var(--color-red)`。

### 5.3. 告警卡片
- **结构：** 背景 `var(--color-surface)`，圆角 `var(--radius-card)`，阴影 `var(--shadow-card)`，左侧边框 `4px solid var(--color-red)`，内边距 `20px`。
- **布局：** 标题行包含图标、标题和严重程度标签；内容描述为辅助文本；底部为操作按钮组。
- **悬停：** 轻微上移 (`transform: translateY(-2px)`)，阴影加强至 `0 8px 25px rgba(0,0,0,0.05)`。

### 5.4. 按钮
- **主要按钮：** 背景 `var(--color-primary)`，颜色白色，圆角 `var(--radius-button)`，内边距 `10px 20px`，字体 `Plus Jakarta Sans` `14px` `600`，无边框。
- **次要按钮（幽灵按钮）：** 背景透明，边框 `1px solid var(--color-border)`，颜色 `var(--color-text)`，内边距 `9px 19px`。
- **悬停：** 主要按钮背景加深（如 `#1E293B`），次要按钮背景变为 `#F8FAFC`。

### 5.5. 筛选栏/分段控件
- **分段控件：** 背景 `#F1F5F9`，圆角 `8px`，内边距 `4px`。每个选项内边距 `6px 12px`，圆角 `6px`，激活状态背景白色并带细微阴影。
- **下拉菜单：** 背景白色，边框 `1px solid var(--color-border)`，圆角 `8px`，内边距 `8px 12px`。

## 6. 图表规范

- **主色：** 柱状图、折线图使用 `var(--color-blue)` (`#3B82F6`)。
- **辅助色：** 对比或负向数据使用 `var(--color-red)`，正向使用 `var(--color-green)`。
- **网格线：** 使用 `1px solid #E2E8F0`，浅色且不干扰数据阅读。
- **工具提示：** 白色卡片样式，圆角 `8px`，阴影，字体 `Albert Sans` `12px`，数值精确。
- **矩形树图：** 区块大小表示预算规模，区块颜色从绿到红渐变表示偏差程度。

## 7. 交互与状态

- **加载状态：** 使用骨架屏，背景 `#E2E8F0`，带动画脉冲效果（`@keyframes pulse`）。
- **空状态：** 居中显示图标和描述文案，搭配引导性按钮（如“同步数据”）。
- **错误状态：** 使用红色 Toast 通知，位置在右上角，带关闭按钮。
- **悬停（可交互元素）：** 指针变为手型，背景或阴影有轻微变化。
- **焦点（键盘导航）：** 外发光 `box-shadow: 0 0 0 2px #3B82F6`。

## 8. 页面布局规格

### 8.1 通用布局规则

#### 等高原则
- **同一行的卡片必须等高**，以底部对齐
- 通过设置相同的 `minHeight` 实现

#### 卡片间距
- 卡片之间间距 `16px`（`--space-16`）
- 行与行之间间距 `20px`

#### 内部滚动
- **触发条件**：当列表内容超过容器高度时
- **行为**：容器内部滚动，不破坏外部布局
- **实现**：
  - 容器设置 `overflow: auto` + 固定高度
  - 表头固定 `position: sticky; top: 0`
  - 最小高度 `200px`，最大高度 `400px`

#### 文字截断
- **表头/标签**：不换行，用 `white-space: nowrap`
- **长文本**：用 `text-overflow: ellipsis` + `overflow: hidden`
- **禁止**：在固定宽度容器内让标签自动换行

## 9. 无障碍规范

### 9.1 触摸目标
- **最小尺寸**：所有可交互元素（按钮、链接、输入框）最小 `44px × 44px`
- **按钮内边距**：`padding: 10px 16px` 确保 44px 最小高度

### 9.2 焦点状态
- **focus-visible**：所有可交互元素必须有可见的焦点环
- **样式**：`outline: 2px solid #3B82F6; outline-offset: 2px`
- **禁止**：`outline: none` 除非有替代方案

### 9.3 颜色对比度
- **正文**：至少 4.5:1（WCAG AA）
- **大文本（18px+）**：至少 3:1
- **UI 组件**：至少 3:1

### 9.4 语义化
- 使用正确的 HTML 标签（`<button>` 而非 `<div onclick>`）
- 表格使用 `<thead>` 和 `<tbody>`
- 图表提供文本替代说明

## 10. 布局代码模式

### 10.1 等高双栏布局（1:2 比例）

```tsx
<div style={{ display: 'flex', gap: 16 }}>
  {/* 左侧 - 1/3 宽度 */}
  <Card style={{ flex: '1 1 33%', minHeight: 320 }}>
    <PieChart />
  </Card>

  {/* 右侧 - 2/3 宽度，内部滚动 */}
  <Card style={{ flex: '1 1 67%', minHeight: 320, display: 'flex', flexDirection: 'column' }}>
    <div style={{ flex: 1, overflow: 'auto' }}>
      <Table>
        <thead style={{ position: 'sticky', top: 0, background: '#fff' }}>
          {/* 表头 */}
        </thead>
        <tbody>
          {/* 数据行 */}
        </tbody>
      </Table>
    </div>
  </Card>
</div>
```

### 10.2 等高双栏布局（1:1 比例）

```tsx
<div style={{ display: 'flex', gap: 16 }}>
```

## 11. 布局检查清单

每次新增或修改看板页面时，检查以下项目：

- [ ] 同一行卡片是否等高？
- [ ] 列表是否设置了内部滚动（而非撑大容器）？
- [ ] 表头是否固定（滚动时不动）？
- [ ] 标签是否设置了 `white-space: nowrap`（避免断行）？
- [ ] 卡片间距是否为 16px？
- [ ] 比例是否符合上述规格？
- [ ] 触摸目标是否 ≥ 44px？
- [ ] 是否有可见的焦点状态？
